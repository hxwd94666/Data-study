## 🎯 面试防御话术提炼 (Interview Focus Areas)
*(请用自己的话，简短盲打回答以下三个你在前两周学到的最核心问题)*

1. **【Python 工程化】** 在处理海量本地日志时，为什么强制要求使用 `with open` 而不是直接 `open()`？它底层解决了什么资源泄露问题？
   - **我的回答**：强制使用with open是为了利用上下文管理器协议，确保文件资源在任何情况下（包括发生异常时）都能被正确、及时地释放。

2. **【SQL 架构优化】** 当使用 `LEFT JOIN` 关联两张表时，如果主表关联键存在海量 `NULL` 值，底层集群会发生什么灾难？如何用 SQL 解决？
   - **我的回答**：如果主表（左表）的连接键存在海量NULL值，这些NULL值的哈希值全部相同，会被分配到同一个Reduce任务（或同一个分区），造成极端的数据倾斜。我们可以使用coalesce加random拼接随机字符串来分类这些key,从而实现分区的均匀分配。

3. **【算法复杂度】** 在 LeetCode 242（字母异位词）中，为什么抛弃 Python 自带的 `.sort()`，转而使用字典（哈希表）来记录频次？
   - **我的回答**：哈希表的时间复杂度是O(n)，而排序的时间复杂度是O(nlogn)，因此在处理海量数据时，哈希表会比排序快很多。
