### 🚀 第一阶段：语言重塑与基础补天（第1-6周，3月中 - 4月底）

**目标：** 彻底掌握Python基础语法，SQL达到面试手撕水平，算法建立初步题感。

- **第1-2周：Python核心语法速通**
  - **学什么：** Python基础数据结构（列表、字典、集合）、函数、面向对象编程（类、继承）。
  - **重点难点：** 列表推导式、**装饰器（Decorator）**、**生成器（Generator）**（这俩面试必问）。
  - **算法/SQL：** 每天力扣（数组、双指针）2题；牛客网SQL必知必会每天3题（温习基础Join和Group by）。
- **第3-4周：Python数据处理利器 + SQL进阶**
  - **学什么：** Pandas 和 Numpy 库的基础操作（DataFrame过滤、合并、分组）。
  - **重点难点（极其重要）：** **SQL窗口函数**（`ROW_NUMBER`, `RANK`, `DENSE_RANK`, `LEAD`, `LAG`）。面试中80%的SQL题都会考窗口函数！
  - **算法：** 字符串、哈希表（每天2题）。
- **第5-6周：Linux进阶与Hadoop基石**
  - **学什么：** 既然你会基础Linux，这周学习Shell脚本三剑客：`grep`, `awk`, `sed`（处理日志极其好用）。
  - **Hadoop理论（不写代码，只懂原理）：** HDFS是怎么存数据的（NameNode/DataNode机制）？YARN是怎么调度资源的？
  - **算法：** 链表、栈与队列。

### 🏗️ 第二阶段：数仓理论与盘活实习经历（第7-12周，5月 - 6月中）

**目标：** 吃透Hive，掌握数据仓库核心概念，**把德勤的实习经历包装成硬核简历亮点**。

- **第7-8周：Hive核心与HQL**
  - **学什么：** Hive底层架构（HQL如何转成MapReduce）。Hive内部表、外部表、**分区表（Partition）**、**分桶表（Bucket）**的区别与应用场景。
  - **实操：** 在虚拟机装一个单机版Hive，自己建表导入数据，写复杂的HQL。

除了保持 GitHub 绿草地，这里再为你深度剖析 4 个能极大拉升秋招竞争力的核心点子：

### 1. 告别“学生味”：养成写企业级规范代码的习惯

面试官看一眼代码，就能大致判断出这是一个“只会做题的学生”，还是一个“可以直接干活的工程师”。很多应届生写的代码能跑通就行，但你可以从第一天起就刻意练习以下三点：

- **强制使用类型提示 (Type Hints)：** Python 是动态类型语言，但企业开发极度看重可维护性。把 `def process_data(data):` 写成 `def process_data(data: list) -> dict:`。
- **戒掉 `print()`，改用 `logging`：** 在处理数据的脚本里，不要到处用 `print()` 打印结果。学着用 Python 内置的 `logging` 模块，输出带有时间戳和级别的日志（如 `INFO`, `ERROR`）。
- **写出人能看懂的 Docstrings：** 为你写的每一个稍微复杂的函数加上规范的注释块，说明这个函数的入参、出参和功能。
- **杀伤力分析：** 当面试官让你手撕代码或看你 GitHub 时，这些规范的细节会极大地掩盖你基础理论起步晚的劣势，展现出极高的职业素养。

### 2. “眼见为实”：用 Streamlit 把项目变成可交互的网页

你的简历上最终会有一个“离线数据仓库”或数据处理的项目。大多数应届生只能在简历上干巴巴地写一行 GitHub 链接，但 HR 或面试官其实很少有时间去下载代码并配置环境运行。

- **怎么做：** 既然你选择了 Python 赛道，后期一定要花一两天时间学一下 **Streamlit**。这是一个极简的 Python 库，能把你的数据处理结果或数据表，直接变成一个漂亮的、可交互的网页仪表盘。
- **杀伤力分析：** 你可以直接把部署好的网页链接写在简历上。面试官点开链接，就能直接看到你清洗好的数据图表。这种“所见即所得”的视觉冲击力，远超千言万语，是绝对的加分项。

### 3. “被看见的努力”：将笔记转化为技术博客输出

与其只在本地写 Markdown 孤芳自赏，不如同步发到掘金、CSDN 或知乎上。

- **怎么做：** 不要写流水账（比如“今天学了列表和字典”），而是写**痛点复盘**。比如未来你学到难点时，写一篇《手把手图解：我是如何彻底搞懂 Hive 数据倾斜问题的》或者《从点鼠标到写代码：深入解析 Airflow 任务依赖配置》。
- **杀伤力分析：** 能够把复杂的技术概念用通俗的语言讲清楚，是高级工程师的必备能力。如果你的文章有哪怕几十个点赞，把博客主页链接贴在简历上，不仅证明了你的学习深度，还能作为面试时的极佳“话语权引导”（面试官大概率会顺着你写的文章来问，这等于你在给他透题）。

### 4. “借船出海”：进行微小的开源贡献 (Open Source)

你目前缺乏竞赛和大型项目，但开源社区向所有人敞开大门。

- **怎么做：** 你的杀手锏是实习期间接触过的 Apache Airflow。它的底层全是 Python。你可以去 GitHub 的 Airflow 仓库里找带有 `good first issue`（适合新手的任务）标签的 Issue，甚至仅仅是去帮官方文档纠正几个错别字、翻译一段最新的英文文档，并提交你的 Pull Request (PR)。
- **杀伤力分析：** 一旦你的代码或文档修改被合并进官方主分支，你的简历上就可以堂而皇之地写上：**“Apache Airflow 开源项目贡献者”**。这几个字的含金量，比你在学校里拿个普通的二等奖学金还要高。
