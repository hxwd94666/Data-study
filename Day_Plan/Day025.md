📅 Day 25：毕设专项 - Amazon 数据集特征工程与推荐模型架构打通
🧱 时间块 1：Amazon 电器/电子数据集特征提取 (3 小时)
【核心任务】：完成从原始 JSON 到模型输入张量（Tensor）的物理映射。

【执行细节】：

类别特征（Categorical Features）：针对 reviewerID 和 asin（商品 ID）进行 Label Encoding 或 Hash Mapping。

数值特征（Numerical Features）：处理商品价格（Price）的缺失值，并进行对数缩放（Log Scaling），缓解长尾分布对 DeepFM 线性部分的影响。

时序特征：提取 unixReviewTime 转化为用户的历史点击序列，为后续双塔模型的“用户塔”提供动态兴趣输入。

【产出】：一份可直接喂入模型的 train_test_features.pkl 文件。

🧱 时间块 2：双塔与 DeepFM 的交互逻辑推演 (3.5 小时)
【核心任务】：编写模型的核心层逻辑，重点解决“高低阶特征交叉”问题。

【执行细节】：

双塔模型（Dual-Tower）：

构建 User Tower（输入：用户画像 + 历史序列）和 Item Tower（输入：商品属性 + 类别）。

实现最后一层的余弦相似度计算，理解其在“大规模在线召回”场景下的检索优势。

DeepFM 模型：

FM 部分：实现二阶特征交叉，捕捉 Amazon 评论中隐藏的商品共现关系。

Deep 部分：构建多层感知机（MLP），提取高阶非线性特征。

Shared Embedding：确保 FM 和 Deep 部分共享同一套 Embedding 向量，减少参数量。

【产出】：核心 Model 类的代码定义。

🧱 时间块 3：实验闭环与指标定义 (1.5 小时)
【核心任务】：定义评估函数，确保推荐效果可量化。

【执行细节】：

离线指标：编写代码计算 AUC（评估点击预测的准确性）和 LogLoss。

推荐排序指标：针对 Top-K 推荐，引入 NDCG@10 和 Recall@10，评估模型在 Amazon 商品海洋中“捞出”用户感兴趣电器的能力。

【产出】：完整的训练与评估 Pipeline 脚本。
