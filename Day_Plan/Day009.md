## ⏰ 详细执行日程与学习闭环 (Execution Plan)

### 🌅 时间块 1：毕设核心代码冲刺与重构 (09:00 - 11:30)
* **【主攻方向】**：推进毕业设计的核心功能模块与数据流转逻辑。
* **【工程化要求】**：抛弃传统学生思维。在 PyCharm 中编写时，若涉及本地文件读取，强制使用 `with open` 上下文管理器；若涉及不可控的数据输入，强制添加 `try...except...finally` 异常捕获机制。
* **【资产沉淀】**：所有产出存入本地 `Grad_Project/` 目录下，确保代码无严重黄线或红线警告。

### ☀️ 时间块 2：组会汇报与需求对齐 (14:00 - 17:00)
* **【主攻方向】**：参与导师组会，进行阶段性成果演示。
* **【职场化演练】**：采用 STAR 法则（情境、任务、行动、结果）进行汇报。重点阐述本周攻克的技术难点，以及通过重构提升了多少代码健壮性。
* **【待办提取】**：精准记录导师提出的修改意见或新增需求，将其转化为下周的开发 Backlog（待办事项）。

### 🌙 时间块 3：复盘规整与规范化提交 (19:00 - 20:00)
* **【代码审查】**：利用 PyCharm 的 Code Inspect 功能，审查今天编写的毕设代码，补齐缺失的函数注释（Google Docstring）。
* **【提交指令】**：在 PyCharm Terminal 中定位到毕设目录，执行 `git add .` -> `git commit -m "feat(grad): 推进毕设核心模块，重构异常处理机制并完成组会节点交付"` -> `git push`。
