## ⏰ 详细执行日程与学习闭环 (Execution Plan)

### 🌅 时间块 1：Python 本地文件流转与模块化 (09:00 - 11:30)
* **【学什么】**：文件的读写模式 (`r`, `w`, `a`)，`with open()` 上下文管理器防止内存泄漏；包与模块的导入 (`import ...`, `from ... import ...`)。
* **【怎么学】**：B 站搜索“Python 文件操作与模块导入”。理解为什么企业级代码绝对不用原生的 `open()` 然后手动 `close()`。
* **【去哪练】**：PyCharm 本地环境造数。在 `First_month/` 目录下手动新建一个 `mock_log.txt`，模拟几行包含 `INFO` 和 `ERROR` 的系统日志。
* **【达标效果与资产沉淀】**：新建 `day08_file_io.py`。要求：写一个函数读取 `mock_log.txt`，利用 `with open` 将所有包含 `ERROR` 的行提取出来，并写入到一个新的 `error_output.txt` 中。必须添加 Type Hint 与 Google Docstring。

### ☀️ 时间块 2：算法突围 - 哈希映射的进阶打击 (14:00 - 15:30)
* **【学什么】**：利用哈希表（Python 字典或定长数组）记录字符出现的频率，实现 O(n) 时间复杂度的词频比对。
* **【怎么学】**：访问“代码随想录”网站，搜索“242. 有效的字母异位词”。理解如何利用 ASCII 码或字典的 Key-Value 结构进行频次加减。
* **【去哪练】**：LeetCode 第 242 题（有效的字母异位词）。
* **【达标效果与资产沉淀】**：坚决不使用极其耗时的 `.sort()` 方法。使用字典统计频次，在 LeetCode 盲打通过后，沉淀至 `First_month/day08_valid_anagram.py`。

### 🗄️ 时间块 3：SQL 跃迁 - CTE 与查询架构解耦 (15:30 - 17:30)
* **【学什么】**：CTE (Common Table Expressions) `WITH ... AS (...)` 的语法规范。深入理解它如何将数百行的嵌套子查询打平成“瀑布流”结构。
* **【怎么学】**：B 站搜索“SQL CTE WITH 用法”或查阅 HiveSQL 官方文档。对比嵌套子查询与 CTE 在可读性和维护成本上的天壤之别。
* **【去哪练】**：本地 PyCharm 新建 SQL 文件进行复杂业务逻辑默写。
* **【达标效果与资产沉淀】**：在 `First_month/day08_cte_refactor.sql` 中，默写一段包含至少三个步骤的 CTE 链路（例如：`WITH step1 AS (...), step2 AS (...) SELECT * FROM step1 JOIN step2...`）。关键字强制全大写，严格首行缩进排版。
