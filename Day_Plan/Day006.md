## ⏰ 详细执行日程与学习闭环 (Execution Plan)

### 🌅 时间块 1：盲盒清零 - Python 匿名函数与高阶应用 (09:00 - 11:30)

* **【学什么】**：`lambda` 表达式（匿名函数）的语法边界，以及它与 `map()`, `filter()` 高阶函数的结合应用。
* **【怎么学】**：阅读 Python 官方文档或检索优质技术博客。理解为什么 `lambda x: x**2` 比 `def` 更适合作为一次性回调函数。
* **【去哪练】**：在 PyCharm 本地环境中编写测试脚本。
* **【达标效果与资产沉淀】**：在 `First_month/` 目录下新建 `day06_lambda_funcs.py`。要求：使用 `filter()` 和 `lambda` 提取出一个列表中的所有奇数。必须添加 Google Docstring 解释代码逻辑。

### ☀️ 时间块 2：算法破局 - 哈希表的降维打击 (14:00 - 15:30)

* **【学什么】**：哈希映射 (Hash Map) 的底层思想（空间换时间）。理解如何利用 Python 的字典 (Dict) 实现 O(1) 复杂度的查找。
* **【怎么学】**：访问“代码随想录”网站，搜索“1. 两数之和”，重点阅读“为什么用哈希表”以及“字典的 key 和 value 分别存什么”的逻辑推演。
* **【去哪练】**：LeetCode 第 1 题（两数之和）。
* **【达标效果与资产沉淀】**：放弃暴力的双重 `for` 循环（O(n²)）。使用 Python 字典实现 O(n) 的解法。在 LeetCode 网页端纯盲打通过后，将代码沉淀至 `First_month/day06_two_sum.py`。

### 🗄️ 时间块 3：SQL 跃迁 - 偏移分析与连续性计算 (15:30 - 17:30)

* **【学什么】**：窗口函数 `LEAD()`（向后取值）与 `LAG()`（向前取值）。
* **【怎么学】**：B 站搜索“SQL LEAD LAG 连续登录”。理解它们在不改变总行数的前提下，如何将上一行或下一行的数据“拉”到当前行进行计算。
* **【去哪练】**：牛客网在线编程 -> SQL篇 -> 检索包含 `LEAD`/`LAG` 的窗口函数真题；或在本地 PyCharm 编写纯 SQL 逻辑推演。
* **【达标效果与资产沉淀】**：在 `First_month/day06_lead_lag.sql` 中默写出基础语法结构：`LAG(字段名, 偏移量, 默认值) OVER (PARTITION BY ... ORDER BY ...)`。关键字强制全大写。
