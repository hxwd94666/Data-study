## ⏰ 详细执行日程与学习闭环 (Execution Plan)

### 🌅 时间块 1：盲盒清零 - Python 模块化与 Shell 日志过滤 (09:00 - 11:30)

* **【学什么】**：Python 模块化导入避免命名冲突；Linux `grep` 命令与管道符 `|` 的组合技。
* **【怎么学】**：B 站搜索“Linux grep 管道符 过滤日志”。理解管道符 `|` 如何将上一个命令的输出作为下一个命令的输入。
* **【去哪练】**：在 Git Bash 或 Linux 虚拟机中，基于 Day 8 生成的 `mock_log.txt`，尝试用命令 `cat mock_log.txt | grep "ERROR"` 过滤错误日志。
* **【达标效果与资产沉淀】**：在 `First_month/` 目录下新建 `day10_shell_grep.sh`，将你能跑通的 Shell 命令记录在注释中。在 Python 侧，新建包目录测试 `import`。

### ☀️ 时间块 2：算法补漏与错题斩杀 (14:00 - 15:30)
* **【学什么】**：清空本周所有的算法技术债。
* **【怎么学】**：回顾 LeetCode 242 (字母异位词) 或 LeetCode 344 (反转字符串)。
* **【去哪练】**：LeetCode 网页端。
* **【达标效果与资产沉淀】**：如果本周有未手写通关的代码，今天必须在 PyCharm 中补齐，并存入 `First_month/`。如果已全通关，尝试用两层 `for` 循环（暴力法）和哈希表法分别跑一次，体会时间复杂度的降维打击。

### 🗄️ 时间块 3：SQL 跃迁 - JOIN 灾难与倾斜预防 (15:30 - 17:30)
* **【学什么】**：各种 JOIN 的图解边界，以及不加过滤条件或关联键包含大量 NULL 值时引发的“笛卡尔积”和“数据倾斜”。 
* **【怎么学】**：B 站搜索“Hive SQL 数据倾斜 JOIN”。理解为什么两个一千万行的表如果用错关联条件，会瞬间产生上百亿行的计算量撑爆集群。
* **【去哪练】**：本地 PyCharm 新建 SQL 文件进行逻辑默写与注释整理。
* **【达标效果与资产沉淀】**：在 `First_month/day10_join_skew.sql` 中编写一段带有详细注释的 SQL，演示如何在 `LEFT JOIN` 前使用 `COALESCE` 处理关联键的 NULL 值以防止倾斜。
