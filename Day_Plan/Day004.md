## ⏰ 详细执行日程与技术路线 (Execution Plan)

### 🧱 时间块 1：算法错题重刷与指法校准 (1.5 Hours)
* **弹药库**：无须看新资料，直接调动大脑里的逻辑缓存。
* **实战靶场**：打开 PyCharm，清空 `First_month/day03_move_zeroes.py` 里的原有代码。
* **技术聚焦**：闭上眼睛回忆“快慢双指针”的运行轨迹。双手放在基准键上，强迫自己用纯盲打敲出完整逻辑。遇到红线报错，优先依靠阅读 IDE 的英文 Error Log 来定位问题。

### 🧠 时间块 2：代码质量扫描与规范重构 (1.5 Hours)

* **实战靶场**：PyCharm `First_month/` 目录下的所有 `.py` 和 `.sql` 文件。
* **技术聚焦**：
  1. Python：检查是否每一个函数都写了明确的入参类型和返回类型定义（如 `-> None:`），补全文档注释。
  2. SQL：把之前挤在一行的 SQL 语句敲回车打散，培养企业级 SQL 的“瀑布流”视觉排版习惯。

### 🗄️ 时间块 3：架构师沉淀报告 (1 Hour)

  2. 窗口函数 `ROW_NUMBER()` 解决的业务痛点是什么。
